# Phase 7 Research — Hybrid Breakthrough Opportunity Search

## What Phase 6 Proved

Phase 6 proved that the bounded hybrid branch is slightly better on the frozen broader paid comparison, but not enough to create a decisive end-to-end leap.

That means the bottleneck is probably not "hybrid retrieval is useless". The more likely problem is that lower-layer gains are not propagating into the final answer path often enough.

## What Is And Is Not Worth Chasing

Worth chasing:
- one or two root mechanisms behind the dominant `prompt_miss / unknown` cluster
- routing/eligibility seams that decide whether a stronger substrate improvement can surface as a final answer win
- local proof shapes that can falsify a breakthrough hypothesis cheaply

Not worth chasing:
- `+1/+2` case-by-case cleanup
- another miniature paid benchmark loop on hand-picked misses
- broad substrate rewrites
- more decision-record prose about baseline vs hybrid selection

## Breakthrough Framing

A good Phase 7 result is not "we fixed some misses". A good Phase 7 result is:

- "here is the dominant blocked path"
- "here is the single best bounded hypothesis"
- "here is the cheapest falsifiable proof"

That turns the next build step into a genuine leverage bet instead of another drift loop.

## Anti-Loop Rules

- Do not run another paid subset inside Phase 7.
- Do not sneak in a per-question patch list.
- Do not treat retrieval-only gain as breakthrough evidence.
- Do not expand Phase 7 into implementation unless the hypothesis is already sharply bounded.

## Likely Root-Mechanism Candidates

The current evidence points most strongly at mechanisms like:

- answer-path eligibility not widening when hybrid retrieval improves
- strict local-comparator / handoff coverage staying too narrow
- routing thresholds keeping too many improved retrieval cases in the same prompt-driven path

Phase 7 should classify these as mechanism classes, not start implementing them all.
