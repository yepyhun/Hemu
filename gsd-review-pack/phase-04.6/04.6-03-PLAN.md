# Plan 04.6-03: Define The Gate Classification Matrix

## Objective

Constrain every remaining live-gate miss to an explicit fail-closed classification so 04.1 stops drifting between kernel bugs and eval artifacts.

## Must Haves

- A small explicit matrix of allowed miss classes
- `unknown` as a required fail-closed bucket
- alignment with existing machine-readable gate status artifacts
- clear rule for when 04.1 should fix kernel code versus handoff/gate interpretation

## Tasks

- Write a short `CORE2-GATE-MATRIX.md` reference artifact.
- Define the allowed classifications:
  - kernel correctness
  - handoff/format
  - judge artifact
  - unknown
- Define what evidence is required before a miss can be assigned to each class.
- Define what follow-up action each class implies:
  - kernel correctness -> fix kernel behavior
  - handoff/format -> fix answer surface/handoff path
  - judge artifact -> record and treat separately from kernel regression
  - unknown -> do not relabel optimistically; gather more evidence
- Align the matrix with the current machine-readable gate artifacts so future reruns can be interpreted against the same contract.
- Hand the phase back to 04.1 with an explicit “fix -> canary -> paid gate” rule rather than open-ended reruns.
- Ensure the rerun rule references the handmade set and golden-path traces as the required local preconditions before the paid gate is retried.

## Verification

- Historical 04.1/04.5 misses can be mapped against the matrix.
- The matrix leaves no “miscellaneous but maybe fine” escape hatch.
- The next 04.1 rerun can be judged against explicit categories rather than prose interpretation.
