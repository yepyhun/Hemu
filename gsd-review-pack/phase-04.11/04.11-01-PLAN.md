# Plan 04.11-01: Lock Validation Protocol

## Goal

Define a fixed broader paid validation protocol for the bounded hybrid branch before any new paid run starts.

## Tasks

1. Define the exact broader paid sample and runtime mode for the hybrid run.
2. Define and write a tiny frozen run manifest:
   - sample size
   - runtime mode
   - benchmark budget tier
   - status artifact path
   - commit/branch marker when available
3. Define the comparison fields that must be reported:
   - pass rate
   - failure-class counts
   - answer-surface hit rate
   - authoritative vs fallback path mix
4. Define the go / no-go / mixed thresholds in advance.
5. Define the verdict precedence rule in advance: regression > mixed > go.
6. Define explicit stop rules that prevent mid-phase implementation drift.
7. Record the protocol as a machine-readable or at least mechanically checkable artifact.

## Verification

- A fixed protocol artifact exists before the paid run.
- A frozen run manifest exists before the paid run.
- The protocol is comparable to the earlier broader evidence.
- The thresholds do not depend on seeing the result first.
