# Plan 04.2-01: Implement Bounded Write-Time Fact Digestion

## Objective

Introduce a narrow digestion layer that runs during `core2_remember(...)` and turn ingestion so common durable memory cases become typed fact records before any later query arrives.

## Must Haves

- Write-time digestion hooks on the existing Core2 write path
- Typed fact candidates for high-value durable cases
- Provenance links back to raw archive or source record
- No new sprawling subsystem outside the existing Core2 boundaries

## Tasks

- Trace the current write path through:
  - `plugins/memory/core2/__init__.py`
  - `agent/core2_runtime.py`
  - `agent/core2_store.py`
- Define a compact fact shape for Phase 04.2, for example:
  - fact kind
  - subject/key
  - normalized value
  - currentness/update markers
  - provenance anchor
- Add a bounded digestion helper that can process:
  - explicit `core2_remember(...)` durable inputs
  - `ingest_turn(...)` inputs after a user/assistant exchange
- Keep the first supported families intentionally narrow:
  - profile/attribute facts
  - explicit update/current/previous facts
  - simple relation facts
- Persist the raw source as before, but also emit smaller typed fact candidates suitable for later canonicalization.
- Ensure the digestion output is inspectable rather than hidden inside opaque text blobs.
- Add targeted tests proving both remember-path and turn-ingest-path inputs produce typed digested records.

## Verification

- Run targeted tests showing durable inputs now create typed fact candidates with provenance.
- Confirm the write path still preserves the raw source while also materializing digested objects.
- Confirm the new layer lives inside the existing Core2 module boundary and does not introduce a second large architecture.
