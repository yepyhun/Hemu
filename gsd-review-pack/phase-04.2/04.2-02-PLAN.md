# Plan 04.2-02: Canonicalize Digested Facts At Write Time

## Objective

Take the new digested fact candidates and resolve them into durable current/previous/update-aware canonical objects at write time, with explicit supersession and relation links.

## Must Haves

- Current vs previous/update semantics resolved during digestion for covered fact types
- Explicit supersession or relation links instead of silent overwrite
- Stable identity for repeated facts so later recall can reason over current truth cheaply
- Continued compliance with existing temporal and provenance rules

## Tasks

- Reuse existing Core2 identity, temporal, and state helpers where possible instead of inventing a parallel rule engine.
- Define how digested facts become canonical truth records:
  - current active fact
  - previous/superseded fact
  - related fact edge
- Implement write-time update handling for covered fact types so a later \"current vs previous\" query no longer depends on reconstructing the timeline from raw conversation text.
- Preserve provenance and link structure, including:
  - `derived_from`
  - `supersedes`
  - relation edges where they are clearly justified
- Ensure the canonicalized result remains explainable via existing inspection surfaces.
- Add tests for:
  - repeated fact updates
  - current/previous transitions
  - preserved history
  - relation linking that does not explode into legacy-style graph sprawl

## Verification

- Run targeted tests proving write-time canonicalization resolves currentness and supersession correctly.
- Confirm earlier truth remains inspectable after a later update.
- Confirm the phase uses the existing Core2 state/edge model rather than hidden implicit overrides.
