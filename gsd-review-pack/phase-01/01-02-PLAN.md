# Plan 01-02: Wire Core2 Through The Existing Hermes Runtime

## Objective

Integrate the now-real Core2 provider into the existing Hermes activation, setup, and lifecycle paths without turning runtime files into Core2 business-logic carriers.

## Must Haves

- Core2 can be selected and initialized through the existing provider activation path
- Runtime and setup surfaces recognize Core2 cleanly
- Lifecycle wiring stays thin and preserves existing builtin + one-external-provider rules

## Tasks

<read_first>

- plugins/memory/__init__.py
- agent/memory_manager.py
- plugins/memory/core2/__init__.py
- plugins/memory/core2/plugin.yaml

<action>
Review the existing external memory activation path in `run_agent.py` and `plugins/memory/__init__.py` and make only the minimal edits needed so Core2 can be discovered, loaded, and registered as the single external provider through the existing `MemoryManager`.

Do not add Core2-specific storage or retrieval logic to `run_agent.py`. Restrict changes there to provider selection, initialization arguments, hook invocation compatibility, and any import-safety fixes required by the existing flow.
</action>

</acceptance_criteria>
</task>
