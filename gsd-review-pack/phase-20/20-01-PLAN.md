# Plan 20-01: Freeze Representative Ten-Case Forensic Slice

## Goal

Freeze one representative fixed `10`-case failing hard subset and define the forensic protocol that will be used to inspect the exact upstream death seam.

## Tasks

- derive the candidate pool from the canonical failed hard cases only
- exclude:
  - already-passed hard cases
  - judge-like artifacts as primary targets
- build a representative fixed `10` with explicit coverage rules:
  - dominant retrieval failures must remain the majority
  - at least `3` operator or question-shape families must be represented if available
  - at most `2` non-retrieval counterexamples may remain, only to prevent tunnel vision
- define one forensic rubric per case:
  - what relevant memory seems to exist
  - what should have been recalled
  - what query/opening path actually ran
  - where the signal appears to die:
    - representation
    - indexing
    - admission
    - query opening
    - pre-selector filtering
- define one canonical comparison rule so every case is analyzed the same way

## Deliverables

- `20-TEN-CASE-MANIFEST.json`
- `20-FORENSIC-RUBRIC.md`
- `20-CASE-LEDGER-SCHEMA.md`

## Verification

- the `10` cases come only from the canonical failed hard set
- the representation rule is explicit, not ad hoc
- the rubric can name a mechanism seam, not just a top-level bucket label
- the subset is frozen before any borrow or code change begins
