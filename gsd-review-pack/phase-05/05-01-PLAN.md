# Plan 05-01: Freeze Comparative Validation Contract

## Goal

Freeze the exact comparison contract for baseline-versus-hybrid validation before any broader execution happens.

## Tasks

1. Define the frozen sample shape, runtime mode rules, status artifact paths, and threshold language.
2. Define explicit verdict categories and precedence rules (`regression`, `mixed`, `promote_candidate` or equivalent).
3. Record anti-drift rules so later execution cannot quietly change the comparison protocol.

## Verification

- The comparison contract is explicit enough that later execution does not need to invent missing rules.

- The protocol forbids hidden implementation work during comparison.
