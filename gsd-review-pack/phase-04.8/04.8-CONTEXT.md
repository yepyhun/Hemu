# Phase 04.8: Generic Surface Generalization - Context

## Phase Boundary

This phase exists because the wider paid `20`-question confirmation run did not fail as random family scatter. The misses clustered around a small number of reusable non-promptless surface gaps:

- generic aggregate totals / thresholds
- generic event ordering / temporal compare
- canonical abstention surfaces

This is a bounded Core2 generalization phase, not a new benchmark loop and not the start of MemPalace adoption.

## Implementation Decisions

### Locked Decisions

- Phase 04.8 is valid only because the `20`-run misses collapse into a few reusable surface patterns rather than into a backlog of topic-specific families.
- This phase must stay inside the Core2 answer-surface/routing layer. It must not reopen promptless-authoritative comparator scope from Phase 04.7.
- Explicitly forbidden in this phase:
  - topic-by-topic family expansion (`charity`, `Spotify`, `Sephora`, `National Geographic`, etc.)
  - comparator expansion or soft semantic “close enough” matching
  - benchmark-specific phrasing fixes
  - broad new write-time extraction families unless strictly required by the existing structured state model
- The deterministic core boundary from Phase 04.6 remains load-bearing:
  - truth/state rules stay small and explicit
  - answer surfaces must fail closed
  - unknowns must not be relabeled into nicer categories
- The rerun discipline remains:
  - fix a specific blocker class
  - local proof
  - canary / failed-cluster proof
  - only then a broader paid rerun
- The routing precedence must stay explicit during execution:
  - existing promptless authoritative surface
  - new generalized non-promptless surface
  - fail-closed fallback
- MemPalace adoption planning is deferred until Phase 04.8 either succeeds cleanly or fails cleanly.

### At the Agent's Discretion

- The exact schema names for the new generalized surfaces may be adjusted if they remain small, inspectable, and obviously reusable.
- Existing `core2_*` modules may be reorganized lightly if it reduces surface/routing confusion without moving truth logic into presentation code.
- The exact local proof shape may mix targeted pytest coverage with small machine-readable artifacts if there is still one authoritative truth source.

## Specific Ideas

- Add a generalized aggregate surface contract that can cover totals and thresholds without topic-specific branching.
- Add a generalized event-order surface contract that can cover pairwise or short-list temporal ordering beyond the current trip-specific path.
- Add a canonical abstention surface for structured “insufficient grounded information” cases so fallback prompt reasoning does not improvise weak non-answers.
- Make the previously failed `20`-run cluster the proof target for this phase, but treat them as exemplars of the three generic patterns rather than as individually special cases.
- Keep a tiny failed-cluster-to-pattern mapping table so each of the `8` previously failed cases is either mapped to one of the three generic patterns or explicitly left out of scope.
