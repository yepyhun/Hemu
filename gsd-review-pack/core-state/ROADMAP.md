#### Phase 23: Bounded Authoritative Surface Payload Expansion

**Goal:** Implement one bounded downstream bridge/payload expansion that converts answer-bearing packets into correct authoritative surfaces on the fixed hard slice, without reopening upstream retrieval or broad architecture work.

**Depends on:** `v1.16/22` verified downstream handoff forensics, `v1.15/21` verified fixed-ten lifecycle forensics

**Status:** Execute complete; fixed-ten paid gate improved the slice to `1/10` but still needs work

**Planned boundaries:**
- allowed:
  - promptless authoritative bridge expansion
  - authoritative payload shaping and correctness fixes
  - answer-surface bridging from already recalled answer-bearing packets
  - fixed-ten local proof and bounded paid replay after local verification
- forbidden:
  - new upstream retrieval heuristics
  - new memory admission/indexing work
  - full prompt assembly redesign
  - MemPalace-style architecture changes
  - ranking/selector reopening without new evidence

**Plans:**
- [x] 23-01 Freeze the bounded downstream fix contract for authoritative surface/payload expansion
- [x] 23-02 Implement the smallest bridge/payload expansion that covers the dominant verified downstream seam
- [x] 23-03 Validate locally, then run the fixed-ten paid gate and record the result honestly
