#### Phase 21: Fixed-Ten Memory Presence And Admission Forensics

**Goal:** Determine, for the fixed representative hard cases, whether the answer-bearing information exists in memory at all, how it was normalized, whether it was indexed, whether any prefilter/admission removed it, and whether handoff formatting loses it.

**Depends on:** `v1.14/20` negative external verdict, fixed-ten forensic artifacts, `v1.10/16` attribution contract

**Status:** Verified

**Planned boundaries:**
- allowed:
  - fixed-case memory presence tracing
  - ingestion / normalization / indexing / admission / handoff inspection
  - old Hermes comparison only as a diagnostic oracle
  - one canonical ledger of where answer-bearing memory dies
- forbidden:
  - new retrieval heuristics by default
  - full legacy import
  - MemPalace-style architecture shift
  - ranking / selector / delivery reopen without forensic proof

**Plans:**
- [x] 21-01 Freeze the exact forensic protocol for the fixed hard slice
