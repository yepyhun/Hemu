# Plan 03-01: Implement Retrieval Routing Matrix And Query-Family Dispatch

## Objective

Replace the current one-route Core2 recall path with an explicit routing layer that classifies query families and executes bounded retrieval plans appropriate to each family.

## Must Haves

- Core2 chooses routes by query family rather than always running the same canonical search
- route selection is explicit and inspectable
- high-risk, exact-source, personal, update-resolution, and exploratory queries no longer collapse into one retrieval path
