# Plan 03-02: Implement Typed Answer Contract, Support Tiers, And Provenance Metadata

## Objective

Turn routed retrieval results into explicit typed answer packets that distinguish exact-source, source-supported, compact-memory, and exploratory behavior with clear provenance and confidence semantics.

## Must Haves

- Core2 returns a typed answer contract rather than only a list of matched records
- support tiers and confidence dimensions are explicit
- provenance and grounding refs are first-class output fields
