# Phase 04.1 Plan 02 Summary

## Accomplishments

- Fixed a real benchmark-seeding correctness bug: LongMemEval benchmark seeds now land in the `personal` namespace instead of `library`, so personal recall routes can actually see them.
- Fixed a real Core2 search bug in `agent/core2_store.py`: lexical scoring no longer overweights one-character substring noise such as `u` in `"The Hate U Give"`.
- Tightened routing in `agent/core2_routing.py` so rewritten reading-history queries stay in `personal_recall` instead of drifting into `factual_supported`.
- Compacted Core2 recall tool payloads via `agent/core2_types.py` and `plugins/memory/core2/__init__.py`, cutting live prompt size dramatically while preserving grounding.
- Replaced the binary benchmark clipping with graded tool-budget tiers (`minimal`, `compact`, `supported`, `full`) so the live gate can spend more evidence budget on comparison and temporal questions without falling back to full raw payloads.
- Added a benchmark-only ephemeral prompt and tiered answer-budget controls in the benchmark harness.

## Issues Encountered

- After correctness fixes, live runs became much more grounded, but answer-generation latency remained dominated by the external model path.
- Graded payload tiers improved prompt pressure and question decomposition, but the live answer path remained somewhat unstable because model reasoning behavior still varied between runs.
