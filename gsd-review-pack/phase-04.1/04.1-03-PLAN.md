# Plan 04.1-03: Re-Run Live Gate And Produce Release Decision

## Objective

Execute the external LongMemEval gate in a staged way after the remaining classified fixes, using the 04.6 axioms, handmade benchmark, golden-path traces, and gate matrix so the result is not another blind paid rerun.

## Must Haves

- Explicit blocker classification before any rerun
- Handmade benchmark and golden-path checks before larger spend
- Small paid or local smoke gate before larger spend
- A staged rerun path up to the larger sample
- Final artifact with pass rate, timing split, token data, and dominant failure patterns
- Explicit go/no-go release recommendation

## Tasks

- Classify the current blocker using `CORE2-GATE-MATRIX.md` before making any new fix or rerun decision.
- If the blocker class is `kernel correctness` or `handoff/format`, implement the targeted fix first.
- Confirm the repaired failure cases stay green against:
  - `CORE2-HANDMADE-BENCH.md`
  - `CORE2-GOLDEN-PATHS.md`
- Define the staged rerun ladder:
  - handmade benchmark
  - golden-path sanity
  - 3-5 item canary/smoke gate
  - intermediate paid subset if the smoke gate is acceptable
  - final larger sample gate
- Run the staged benchmark on the current Core2 kernel only after the local preconditions are green.
- Stop early if the latency or failure profile clearly violates the gate.
- Capture:
  - pass rate
  - dominant failure categories
  - average and worst timing split
  - token savings versus replay baseline
  - whether the remaining bottleneck is retrieval, answer generation, judge cost, wrapper overhead, or unresolved correctness gaps
- Write a release-decision artifact under the phase directory with a plain conclusion.
- When a miss survives into the paid gate, classify it again against the gate matrix rather than relabeling it narratively.

## Verification

- Confirm the handmade benchmark and golden-path traces are green before the paid rerun step.
- Run the staged LongMemEval benchmark through the benchmark script using the real Core2 Hermes path.
- Produce inspectable JSON or markdown artifacts under the phase directory.
- Confirm the release recommendation matches:
  - the 04.6 axioms
  - the handmade benchmark
  - the golden-path traces
  - the observed live gate evidence
