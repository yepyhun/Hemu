# Plan 04.3-02: Wire Runtime Recall To Prefer Facts First

## Objective

Teach Core2 runtime recall and the authoritative path to use fact-first candidates for covered query families, with safe fallback to broader canonical retrieval only when fact evidence is insufficient.

## Must Haves

- Covered query families prefer fact-first candidates
- Safe fallback to broader canonical retrieval remains intact
- Current/previous/update semantics continue to work correctly
- Authoritative answer path benefits from the same fact-first packet flow

## Tasks

- Decide which query families are covered in Phase 04.3, keeping the set intentionally narrow.
- Update route/runtime behavior so covered queries try fact-first candidate retrieval before generic `search_canonical(...)`.
- Define clear fallback rules when:
  - no matching fact exists
  - only partial fact evidence exists
  - broader grounded evidence is still required
- Keep the existing abstention and provenance discipline unchanged or stronger.
- Align `agent/core2_authoritative.py` and the provider's direct-answer path with the same fact-first packets where it is a net win.
- Avoid universal rerouting; leave unsupported query families on the existing path.

## Verification

- Run targeted tests proving covered queries actually prefer the fact substrate first.
- Confirm raw/canonical fallback still works when the fact substrate is insufficient.
- Confirm authoritative answers on covered cases remain grounded and do not regress into hidden heuristics.
