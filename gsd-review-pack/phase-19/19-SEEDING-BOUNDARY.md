# Phase 19 Seeding Boundary

## Included In This Phase

- one bounded decomposition layer before selector engagement
- one bounded operator-slot schema
- one bounded rewrite family per admitted operator family
- one bounded set of seed families:
  - temporal anchor pair
  - aggregate operand seed
  - previous/current seed
  - pairwise comparison seed
- route and trace observability for schema detection and seed opening

## Explicitly Excluded

- full semantic parsing
- full staged retrieval or hierarchical retrieval adoption
- context-tree or provider-style platform rewrite
- selector logic changes
- delivery path changes
- bitemporal truth semantics
- truth-discovery weighting
- aggregation safety redesign
- calibrated override gating

## Why The Boundary Is Still Narrow

The phase got larger than the original minimal-seed idea, but it is still one seam:

- upstream of selector engagement
- bounded to one frozen family
- bounded to four operator families
- bounded to rewrite-and-seed primitives only
