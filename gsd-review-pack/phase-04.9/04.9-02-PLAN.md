# Plan 04.9-02: Build The Adoption Matrix And Layer Ownership Map

## Objective

Decide, for each comparison layer, whether Core2 should adopt MemPalace as-is, adapt selected ideas into Core2, keep the current Core2 approach, or leave the verdict undecided.

## Must Haves

- One row per stable layer from Plan 04.9-01.
- One explicit verdict per row:
  - `adopt as-is`
  - `adapt into Core2`
  - `keep Core2`
  - `undecided`
- Single-owner rule for every layer.
- Complexity risk noted for each row.

## Tasks

1. Write `MEMPALACE-ADOPTION-MATRIX.md` with columns:
   - layer
   - current Core2 approach
   - MemPalace approach
   - verdict
   - rationale
   - evidence strength
   - complexity risk
2. Write `LAYER-OWNERSHIP.md` that makes explicit:
   - which layers remain Core2-owned
   - which layers could become MemPalace-owned
   - which layers would only ever be hybrid adapters
3. Make the critical boundaries explicit:
   - no dual truth engine
   - no silent benchmark-story substitution for answer correctness
   - no substrate rewrite before ownership is locked

## Verification

- Every layer has one explicit verdict.
- Every layer has one owner.
- The matrix contains at least one row in each meaningful category if the evidence supports it; otherwise, it says `undecided` rather than forcing symmetry.
