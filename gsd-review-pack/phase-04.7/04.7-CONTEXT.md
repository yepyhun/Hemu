# Phase 04.7: Authoritative Gate Compatibility - Context

## Phase Boundary

Phase 04.7 exists to close the remaining external gate mismatch around promptless authoritative answers before Phase 04.1 spends the next paid rerun. The phase is intentionally narrow: the latest local and canary evidence indicates the covered Core2 kernel path is producing direct authoritative answers, but the external gate still treats some promptless short-circuit cases as `judge: unknown` or ordinary prompt misses.

This phase is not allowed to reopen kernel family expansion, retrieval-contract redesign, or benchmark-specific phrasing inside the deterministic core. Its job is to normalize the gate-facing compatibility path for answers that already come from the provider-owned structured surface.

## Implementation Decisions

### Locked Decisions

- Treat the remaining promptless-authoritative mismatch as evaluator/gate compatibility work, not as evidence that new durable-memory families are missing.
- Keep the deterministic core boundary from Phase 04.6 intact:
  - no benchmark-specific English phrasing inside `core2_authoritative.py`
  - no moving truth/state logic into the serializer or gate runner
- The canonical comparator must stay strict and deterministic:
  - no soft semantic “close enough” matching
  - no pass when structured payload and rendered answer meaning diverge
- Prefer a deterministic local comparator/canonicalization path before trusting the external evaluator in promptless-authoritative cases.
- Record promptless authoritative short-circuit cases explicitly in gate artifacts so they cannot be misread as ordinary prompt misses.
- Use canary-first proof before the next paid rerun.
- Canary green alone is not enough to justify the next paid rerun; the specific previously failing promptless-authoritative cases must be reclassified or resolved explicitly.

### At the Agent's Discretion

- Exact artifact names and helper function names
- Whether the local comparator lives beside the benchmark runner or in a small dedicated helper
- Exact shape of canonical normalization so long as it remains deterministic, small, and inspectable

## Specific Ideas

- Add a deterministic local comparator for promptless authoritative answers before the external evaluator is treated as final.
- Mark promptless authoritative short-circuit cases explicitly in run artifacts.
- Fail closed on ambiguous/malformed judge responses, but keep them distinguishable from kernel-correctness failures.
- Spend canary proof first; only then justify the next paid rerun in 04.1.

## Deferred Ideas

- Do not redesign the benchmark stack as a whole.
- Do not add new durable-memory families here.
- Do not move linguistic styling back into the deterministic core.
- Do not turn 04.7 into a broad artifact-refactor phase; only tighten the authoritative source of gate truth for the current work.
