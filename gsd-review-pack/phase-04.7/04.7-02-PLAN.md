# Plan 04.7-02: Harden Gate Runner Classification For Evaluator-Side Failures

## Objective

Ensure the external gate runner records promptless-authoritative short-circuit cases explicitly and fails closed on evaluator ambiguity without disguising it as kernel breakage.

## Must Haves

- Explicit promptless-authoritative markers in artifacts
- Distinguishable `judge artifact` / evaluator-side failure classification
- Fail-closed handling for malformed or ambiguous judge results
- No reopening kernel family or retrieval scope
- One authoritative machine-readable gate-status source for the active 04.1/04.7 compatibility path

## Tasks

- Extend gate-run artifacts so promptless authoritative short-circuit cases are explicit in the status output.
- Pick and document the single authoritative gate-status file for this compatibility path; other docs should derive from it instead of competing with it.
- Make the runner preserve the difference between:
  - local comparator success/failure
  - external judge ambiguity
  - real handoff/format mismatch
- Ensure malformed or missing judge outputs fail closed but remain visible as evaluator-side issues.
- Update or add targeted tests around the gate runner so the classification rules are mechanically checked.
- Keep the resulting artifact compact and readable enough for manual inspection.

## Verification

- Machine-readable gate artifacts explicitly show promptless-authoritative short-circuit cases.
- `judge: unknown` cases are not silently collapsed into kernel misses or generic prompt misses.
- Gate-runner tests covering the new classification behavior are green.
- The active gate truth is readable from one primary status artifact without needing to reconcile multiple competing status files.
