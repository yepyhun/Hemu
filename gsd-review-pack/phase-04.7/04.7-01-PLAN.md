# Plan 04.7-01: Add A Canonical Local Comparator For Promptless Authoritative Answers

## Objective

Give promptless authoritative answers a deterministic local comparability path so the gate can evaluate them mechanically before an external judge response is treated as authoritative.

## Must Haves

- Small, deterministic normalization/comparison logic
- Scoped first to promptless authoritative answers
- No benchmark-specific phrasing in the deterministic core
- Inspectable result that can be recorded in artifacts
- No fuzzy semantic pass condition
- No pass when structured payload and rendered answer disagree materially

## Tasks

- Identify the exact answer shapes that currently produce `judge: unknown` despite `answer_surface_hit: true` and `api_calls: 0`.
- Add a small canonical comparator/helper around the gate path for promptless authoritative answers.
- Make the comparator operate on structured/canonical answer content rather than raw benchmark-specific English phrasing whenever possible.
- Add an explicit coherence rule between structured payload and rendered answer so comparator success cannot hide a semantically degraded surface.
- Record comparator outcomes in the machine-readable gate status so later phases can distinguish:
  - comparator pass
  - comparator fail
  - comparator not applicable
- Add focused tests for comparator behavior using current failing canary-style examples.
- Add at least one negative proof where a superficially similar answer must still fail.

## Verification

- Targeted benchmark/comparator tests are green.
- Current promptless-authoritative examples are classified deterministically rather than falling immediately to ambiguous `judge: unknown`.
- No new benchmark-friendly phrasing is introduced into `core2_authoritative.py`.
- Comparator success cannot be produced by loose “close enough” matching.
