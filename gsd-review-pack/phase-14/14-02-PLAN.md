# Plan 14-02: Implement Budgeted Selector And Aggregation Safety

## Goal

Implement one bounded set-selection layer plus narrow aggregation safety inside the existing retrieval seam.

## Tasks

- add a budget-aware selector that scores candidate sets through slot/operator sufficiency rather than itemwise pickup alone
- wire the selector only into the bounded supported shapes
- add narrow aggregation safety checks for dedupe and compatibility
- make partial-evidence composition abstain instead of overcommitting
- keep explicit observability around selector route-shifts and safety-triggered abstention

## Deliverables

- bounded retrieval-side code changes
- targeted tests for selector behavior and aggregation safety

## Verification

- supported shapes use the bounded selector path
- excluded shapes remain excluded
- unsafe aggregation paths abstain instead of composing silently
- no truth ownership, ranking, or delivery drift appears

## Guardrails

- no generic selector framework for the whole kernel
- no broad feature explosion
- no current-total aggregate reopening
