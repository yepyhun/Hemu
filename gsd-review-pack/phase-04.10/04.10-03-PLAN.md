# Plan 04.10-03: Prove Non-Regression And Define Cutover Rules

## Objective

Prove that the bounded hybrid substrate improves the lower layer without regressing Core2 deterministic behavior, then define whether cutover is justified.

## Must Haves

- local proof for substrate and retrieval
- regression proof for Core2 truth/answer behavior
- explicit cutover criteria
- explicit stop rule if ownership or behavior becomes ambiguous

## Tasks

1. Add or extend targeted tests around:
   - raw/verbatim substrate behavior
   - scoped retrieval behavior
   - Core2 truth/answer non-regression
2. Record one or two paired retrieval traces:
   - old path
   - new hybrid path
   for the same canonical queries
3. Re-run the local proof suites most relevant to:
   - truth/state
   - answer surfaces
   - abstention
4. Write the cutover decision rule:
   - when the hybrid substrate is good enough to become default
   - when it should stay shadow/optional
   - when the phase should stop
5. Define a clean rollback switch rule so the old path can be restored if the hybrid path regresses behavior or debuggability.
6. Define the next-step handback:
   - broader paid rerun
   - or another bounded substrate phase
   - but not open-ended expansion

## Verification

- Non-regression against Core2 hard-core behavior is explicit.
- Cutover criteria exist and are fail-closed.
- Rollback behavior is explicit.
- Paired retrieval traces exist for debugging and review.
- The handback does not assume endless substrate expansion.
