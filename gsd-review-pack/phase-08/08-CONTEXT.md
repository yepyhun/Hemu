# Phase 8: Authoritative Eligibility Bridge - Context

**Gathered:** 2026-04-07
**Status:** Ready for planning
**Source:** milestone carry-forward from archived `v1.1`

## Phase Boundary

Phase 8 builds one bounded routing/eligibility bridge so that stronger hybrid evidence for **already-covered families** reaches the existing deterministic authoritative answer path more often.

This phase is **not**:
- a new retrieval/substrate phase
- a comparator expansion phase
- a handoff-format cleanup phase
- a broad benchmark rerun phase

## Locked Decisions

### Primary Target
- The only breakthrough target carried forward from `v1.1` is the **authoritative-eligibility bridge for already-covered families**.
- The bridge must operate at the routing/eligibility seam, not by changing truth/state ownership.

### Explicit Exclusions
- No new family proliferation.
- No comparator softening or broad comparator growth.
- No benchmark-specific phrasing.
- No broad paid rerun inside this phase.
- No reopening the MemPalace/substrate adoption question.

### Proof Shape
- The bridge must be tested on a **small frozen blocked-path slice** drawn from the unchanged `prompt_miss / judge: unknown` cluster.
- The proof must show a **route-shift** into the existing authoritative/deterministic path, not just better retrieval traces.
- The phase must emit a **single canonical local outcome file** so the next step is decided from one truth source.
