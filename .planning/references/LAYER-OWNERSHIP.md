# Layer Ownership

## Purpose

Single-owner map for any future Core2 / MemPalace hybrid.

Rule: one layer, one owner.

## Ownership Map

| Layer | Owner | Notes |
|---|---|---|
| Deterministic truth / state / provenance | Core2 | Non-negotiable hard-core ownership |
| Deterministic answer surface / abstention | Core2 | Must stay fail-closed and Core2-owned |
| Benchmark / evaluation harness | Core2 | MemPalace benchmark materials are comparative input, not the final gate |
| Lossless raw substrate | Hybrid adapter under Core2 governance | Likely MemPalace-inspired implementation, but not independent ownership |
| Ingest / routing substrate | Hybrid adapter under Core2 governance | May borrow MemPalace chunking/routing ideas before Core2 digestion |
| Semantic retrieval / scoped filtering | Hybrid adapter under Core2 governance | Strongest likely borrowing seam |
| Wake-up / compression | Hybrid adapter or optional layer | Useful, but not allowed to redefine the truth core |
| Graph / navigation | Optional secondary layer | Never primary truth owner |
| Temporal knowledge graph | Optional secondary layer | May support retrieval/evidence, but not Core2 truth replacement |
| MCP / operator surface | Mixed / adapter layer | Ergonomic borrowing is allowed; truth ownership is not |

## Forbidden Ownership States

- Core2 truth/state + MemPalace truth/state both active as peers
- MemPalace graph treated as authoritative truth while Core2 supersession/state logic also remains active
- Wake-up/bootstrap layer silently redefining the answer surface
- Benchmark recall numbers replacing answer-correctness gates

## Practical Rule

If a future phase cannot state one owner for a layer in one sentence, that phase is too complex and should be rejected or split.
