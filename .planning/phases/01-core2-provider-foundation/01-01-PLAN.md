# Plan 01-01: Make The Core2 Provider Stub Real

## Objective

Turn the existing Core2 provider stub into a loadable, import-safe, locally persistent provider foundation with an explicit runtime module boundary. This plan exists to eliminate the current “plugin exists but runtime import is missing” gap.

## Must Haves

- Core2 provider imports cleanly through the existing plugin path
- `agent.core2_runtime.Core2Runtime` exists and supports the provider methods currently called
- Core2 foundation logic is isolated into dedicated modules rather than inflating `plugins/memory/core2/__init__.py`
