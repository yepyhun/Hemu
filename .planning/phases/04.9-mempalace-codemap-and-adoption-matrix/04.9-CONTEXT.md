# Phase 04.9: MemPalace Codemap And Adoption Matrix - Context

## Phase Boundary

Phase 04.9 is a decision and codemap phase, not an implementation phase.

Its job is to compare Core2 and MemPalace layer-by-layer, then produce an adoption matrix that says which pieces should be:

- adopted as-is
- adapted into Core2
- explicitly kept Core2-native

This phase must not start substrate rewrites, benchmark reruns, or wholesale replacement work.

## Implementation Decisions

### Locked Decisions

- 04.9 exists to prevent blind replacement, not to justify it.
- The comparison must be layer-by-layer, not repo-vs-repo in the abstract.
- The minimum comparison ontology for this phase is:
  - lossless raw storage / verbatim substrate
  - ingest / chunking / routing
  - semantic retrieval / scoped filtering / rerank
  - wake-up / compression / bootstrap context
  - graph / navigation
  - temporal knowledge graph
  - deterministic truth / state / provenance
  - deterministic answer surface / abstention
  - benchmark / evaluation harness
- One layer must have one owner. 04.9 must not authorize two competing truth engines.
- MemPalace retrieval claims are meaningful input, but they are not by themselves proof that MemPalace is a better end-to-end memory kernel than Core2.
- The end-state after 04.9 may be:
  - keep Core2 mostly as-is
  - build a bounded hybrid
  - replace one bounded layer
  - but not an assumed wholesale repo switch
- If evidence for a layer is weak or ambiguous, the matrix must say `undecided` rather than forcing a neat answer.

### At the Agent's Discretion

- Exact names for the comparison layers, if they stay stable and intuitive.
- Whether a single adoption matrix is enough, or whether a second ownership table makes the result clearer.
- Whether a broader paid rerun is recommended before or after the first MemPalace-derived implementation phase, as long as the recommendation is explicit and conditional.

## Specific Ideas

- MemPalace currently looks strongest in:
  - verbatim storage
  - scoped retrieval
  - wake-up layering / compression
  - optional navigation / graph substrate
- Core2 currently looks strongest in:
  - deterministic truth/state boundaries
  - provenance and supersession discipline
  - fail-closed answer surfaces
  - abstention guarantees
- The most likely good outcome is not `Core2 or MemPalace`, but a bounded hybrid with explicit ownership.

## Deferred Ideas

- Any actual substrate swap
- Any AAAK rollout inside Core2
- Any manual KG protocol becoming the Core2 truth engine
- Any benchmark-led “winner” claim before the adoption matrix is complete
