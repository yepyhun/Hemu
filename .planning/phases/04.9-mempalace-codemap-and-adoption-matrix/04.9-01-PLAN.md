# Plan 04.9-01: Codemap MemPalace And Normalize The Comparison Layers

## Objective

Create a concrete MemPalace codemap that is comparable to Core2 layer-by-layer instead of repo-by-repo.

## Must Haves

- Inventory the MemPalace modules relevant to memory behavior.
- Group them into a stable comparison ontology shared with Core2.
- Distinguish code-anchored evidence from README-only claims.
- Keep the codemap small enough to guide decisions rather than becoming a second documentation project.

## Tasks

1. Read the MemPalace sources needed to understand:
   - ingest / chunking / routing
   - storage / retrieval
   - wake-up / compression
   - graph / KG
   - MCP/tool surface
   - benchmark harness
2. Write a compact reference document:
   - `MEMPALACE-CODEMAP.md`
3. Normalize the comparison layers into a shared ontology that will be used by the adoption matrix:
   - substrate
   - retrieval
   - wake-up / compression
   - graph / KG
   - truth / state
   - answer / handoff
   - benchmarking
4. Record for each layer:
   - concrete module anchors
   - what the layer appears to do
   - what evidence strength we actually have

## Verification

- `MEMPALACE-CODEMAP.md` exists.
- Each major MemPalace layer has at least one concrete code anchor.
- The comparison ontology is stable enough to reuse in later plans without rewriting it.
