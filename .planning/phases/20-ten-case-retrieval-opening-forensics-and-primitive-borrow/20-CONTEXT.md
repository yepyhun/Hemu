# Phase 20 Context

## Why This Phase Exists

`v1.13/19` proved a larger bounded upstream decomposition-and-seeding stack locally, but the follow-up random paid hard sample stayed at `0/5`.

That means:

- the local proof was real
- the active hard wall still did not move
- the next step must not be another elegant broad build

The correct next move is a stricter compromise:

- inspect a representative fixed `10`-case failing subset deeply enough to find the exact upstream death seam
- borrow at most one narrow primitive family if the codemap shows a genuine fit
- test that bounded borrow on the same fixed `10` cases

## Hard Truth To Preserve

- the canonical hard residual replay remained weak externally
- the random paid hard sample after `v1.13` was `0/5`
- the system still shows dominant upstream failure before selector engagement on the hard residual path
- Phase 20 must not hide behind local-only green tests

## What Counts As Progress

This phase is only useful if it produces one of:

- an exact named upstream death seam that explains a large part of the fixed `10`
- a bounded primitive borrow that measurably moves the fixed `10` replay
- a clean stop verdict that prevents more zombie complexity

## Borrow Policy

Allowed inputs:

- old Hermes query-signal helpers
- old Hermes temporal/current helpers
- old Hermes unit/scope primitives
- external systems only as narrow design inspiration

Forbidden inputs:

- full legacy projection-system import
- MemPalace-style architecture adoption
- full state-semantics rewrite
- broad retrieval redesign
- selector, delivery, or ranking reopen
