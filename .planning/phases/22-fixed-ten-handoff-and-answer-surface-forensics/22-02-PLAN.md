# Plan 22-02: Trace Packet, Surface, Payload, And Handoff Loss Seams

## Goal

Trace the fixed ten through the downstream chain and assign each case to one concrete seam using captured artifacts rather than intuition.

## Tasks

- capture direct recall packet properties that matter for downstream answerability
- record whether a structured answer surface is built, and if not, why it falls back
- record whether authoritative payload resolution succeeds, and if not, which resolver family blocks it
- compare locally available answer-bearing content against the prompt/handoff material seen by the final agent path
- record whether the final response actually surfaces the same grounded answer-bearing material
- produce one per-case ledger and one aggregate seam taxonomy

## Deliverables

- `22-DOWNSTREAM-LEDGER.jsonl`
- `22-SEAM-TAXONOMY.md`
- `22-PACKET-HANDOFF-COMPARATOR.md`

## Verification

- every fixed-ten case receives one dominant downstream seam label or an explicit residual label
- the ledger cites concrete checkpoint evidence, not only summary prose
- the comparator shows the difference between local packet evidence and final handoff evidence where relevant
- no new upstream retrieval hypothesis is introduced unless the trace directly falsifies the downstream premise
