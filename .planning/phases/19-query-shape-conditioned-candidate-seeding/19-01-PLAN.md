# Plan 19-01: Freeze Target Subset And Operator-Slot Schema

## Goal

Freeze the exact hard-replay subset where decomposition-driven query-shape seeding is still a plausible seam, and define the operator-slot schema plus engagement criterion for that slice.

## Tasks

- derive the target subset from canonical `v1.12/18` taxonomy plus the paid hard replay attribution artifacts
- exclude cases that are already clearly:
  - unsupported upstream shapes
  - entity/scope binding misses
  - judge-like artifacts
  - downstream delivery failures
- formalize one bounded schema for the frozen subset:
  - operator family
  - arity
  - required slots
  - dominant constraints:
    - temporal
    - scope
    - distinctness
- define one explicit engagement criterion for the phase:
  - movement in `evidence_present_cases`, or
  - movement in `sufficient_retrieval_cases`, or
  - explicit upstream seed-opening traces on the target subset
- write one canonical seam-rules document that states what counts as engagement and what counts as immediate falsification

## Deliverables

- `19-TARGET-SUBSET-MANIFEST.json`
- `19-OPERATOR-SLOT-SCHEMA.md`
- `19-SEEDING-SEAM-RULES.md`

## Verification

- target subset is derived only from canonical `v1.12/18` and `v1.13` paid replay artifacts
- every target case remains upstream of selector engagement
- every target case is still compatible with one operator-slot family rather than a broad retrieval rewrite
- the engagement criterion is measurable from artifacts, not from prose interpretation
