# Phase 04.3 Plan 01 Summary

## Accomplishments

- Added narrow fact-first retrieval primitives on top of the Phase 04.2 substrate instead of inventing a second retrieval architecture.
- Core2 can now look up covered digested facts by `fact_key` through `agent/core2_store.py`.
- The runtime gained explicit covered-query detection for the first fact-first families:
  - current occupation
  - previous occupation
  - current residence
  - current timezone
  - current manager/team
- Fact-first hits are now inspectable through record metadata and route notes instead of being hidden inside generic ranking.

## Files Created/Modified

- `agent/core2_store.py`
- `agent/core2_runtime.py`

## Issues Encountered

- Personal-record prioritization still favored a noisy raw note over the fact-first candidate until the ranking was updated to recognize explicit fact-first markers.
- The Phase 04.2 digestion path had a real title bug: remember-ingested fact records reused the parent note title instead of the fact title. This was fixed during 04.3 execution because it directly affected proof quality.
