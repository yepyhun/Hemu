# Plan 04.6-02: Create The Handmade Acceptance Set

## Objective

Create a small handmade benchmark that proves “we are walking in the right direction” before spending the final external gate again.

## Must Haves

- 8-12 real memory cases
- 1-2 cases per important covered family
- explicit expected outcome per case
- explicit pass/fail meaning
- not primarily judged by another LLM

## Tasks

- Write a short `CORE2-HANDMADE-BENCH.md` reference artifact.
- Add one or two tiny `golden path` traces that show the intended good path from ingest through structured facts to final answer surface.
- Choose a small case set that covers the important durable-memory families and kernel behaviors, such as:
  - stable personal attribute recall
  - preference/habit recall
  - collection/count update
  - temporal update/currentness
  - conflict or abstention
  - one handoff-sensitive case
- For each case, specify:
  - setup or prior remembered facts
  - the query
  - the expected answer or abstention
  - the expected structured support/path
  - what counts as fail
- Decide whether a tiny machine-readable fixture should accompany the markdown spec for repeatable local execution.
- Keep the set small enough to be run and interpreted frequently by a human without benchmark fatigue.
- Define a paid-gate rerun trigger rule: the external paid gate should only run after the handmade set is green and the change under test is tied to an explicit blocker class.

## Verification

- The handmade set can be read and judged by a human without extra context.
- The cases are concrete enough that “pass” and “fail” are not ambiguous.
- The set is clearly positioned as a pre-gate directional proof, not a replacement for the final external benchmark.
- The rerun trigger rule is explicit enough to prevent another benchmark-first patch loop.
