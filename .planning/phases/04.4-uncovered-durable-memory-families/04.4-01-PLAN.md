# Plan 04.4-01: Extend Write-Time Coverage For Uncovered Durable Families

## Objective

Make the uncovered durable-memory families first-class Core2 substrate citizens at write time instead of leaving them for raw query-time reconstruction.

## Must Haves

- Preference/habit/routine memory gets typed fact coverage
- Aggregate/count memory gets typed fact or summary coverage
- Temporal-summary/event-anchor memory gets typed fact coverage
- Provenance, supersession, and inspectability remain intact

## Tasks

- Extend `agent/core2_fact_registry.py` with new covered family specs for:
  - preference/habit/routine
  - aggregate/count
  - bounded temporal summary/event anchors
- Extend `agent/core2_digestion.py` so `remember(...)` and turn ingestion can produce compact typed candidates for those families.
- Canonicalize the new family records using the existing Core2 state model:
  - current vs overridden preference
  - aggregate/count dedupe and scope
  - temporal anchor/event identity
- Materialize compact retrieval-ready and/or delivery-ready artifacts for the new families rather than leaving them implicit in raw notes.
- Keep the new coverage narrow and explicit. Do not add broad semantic interpreters or a second graph subsystem.

## Verification

- Add or extend targeted tests proving the new fact families are created at write time.
- Show that provenance and currentness are inspectable for the new records.
- Confirm existing covered families remain unaffected.
