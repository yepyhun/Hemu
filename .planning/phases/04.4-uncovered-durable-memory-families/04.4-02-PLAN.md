# Plan 04.4-02: Consume The New Families Through Fact-First Recall

## Objective

Make runtime recall and authoritative answering consume the new durable-memory family artifacts directly, with bounded fallback only when those structured artifacts are insufficient.

## Must Haves

- The new families are reachable through fact-first recall
- Authoritative answers read from structured packet content, not raw-language heuristics, for these families
- Fallback remains safe and provenance-preserving
- Fast-answer behavior for these families is explicitly guarded so it only fires from structured fact-first evidence

## Tasks

- Extend `agent/core2_runtime.py` and related store helpers so the new covered families can be matched and retrieved fact-first.
- Ensure packet metadata stays inspectable for the new families:
  - `fact_key`
  - retrieval path
  - compact summary or support artifacts
- Extend `agent/core2_authoritative.py` so preference, aggregate/count, and bounded temporal-summary answers resolve from structured facts and compact views where possible.
- Add an explicit guard for the new families so a direct answer only resolves when packet metadata shows a structured path such as `fact_key`, `fact_first`, or an equivalent compact structured artifact.
- Keep broader canonical fallback when structured evidence is insufficient, but do not let the new families fall back immediately to generic broad reasoning.
- Keep the provider and benchmark entry surfaces thin; the new behavior should be a Core2 runtime capability, not a benchmark-only branch.

## Verification

- Add focused tests for fact-first recall and authoritative answering on the new families.
- Prove that raw heuristics are no longer the primary path for those families and that the new guard blocks unstructured fast-path activation.
- Confirm fallback still preserves abstention and correctness.
