# Phase 04.12: Residual Gate Closure - Research

## User Constraints

- The user wants no infinite repair loop.
- The user wants the remaining work kept narrow and honest.
- The user does not want the project to silently reopen broader architecture after a `go` verdict.

## What 04.11 Actually Authorized

Phase 04.11 authorized one more bounded follow-on phase because the hybrid branch clearly improved:

- pass rate
- answer-surface hit rate
- promptless authoritative coverage
- wall-time cost

It did not authorize:

- a new substrate branch
- broad compatibility work
- another architectural comparison cycle

## Recommended Closure Shape

Treat the two residuals as two different bounded closure problems:

1. `handoff_format` residual:
   - fix only if a narrow canonical handoff improvement exists
2. `judge_artifact` abstention residual:
   - preserve semantic honesty first
   - only normalize if it stays inside the fail-closed abstention contract

The phase should distinguish “actually fixable residual” from “acceptable evaluator artifact that should stop here”.

## Best 3-Plan Split

### Plan 04.12-01

Reproduce and classify the two residuals with explicit protocol.

### Plan 04.12-02

Implement only the narrow residual closure that stays within current ownership boundaries.

### Plan 04.12-03

Re-run the residual proof, then close or explicitly stop.

## Testing Guidance

- Re-run only the residual cases first, not a broader paid set.
- Preserve the 04.11 canonical protocol assumptions where relevant.
- Require explicit evidence for whether the abstention residual is fixable versus merely judge-noisy.
- If the handoff residual fix widens beyond narrow serialization/handoff behavior, stop.

## Risks And Failure Modes

- treating a judge artifact like a kernel bug
- softening abstention just to satisfy a judge
- reopening substrate scope because one residual remains annoying
- turning two residual cases into another generic expansion phase

## Open Questions

- Can the handoff-format miss be fixed through a narrow serialization/handoff rule only?
- Is the abstention residual truly closable, or is it a legitimate stop-point under fail-closed semantics?
- If one residual stays open, what exact closure language should the phase use so the roadmap does not drift?
