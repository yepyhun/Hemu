# Plan 23-01: Freeze The Authoritative Surface Contract

## Goal

Freeze the exact supported-family boundary and bridge contract before implementation so the phase cannot silently widen into a general downstream rewrite.

## Tasks

- reuse the same fixed-ten manifest and the verified Phase 22 seam counts
- define which answer-bearing packet shapes are in scope for promptless authoritative bridging
- define what a correct authoritative surface must contain for each supported family
- define explicit out-of-scope families that must continue to fall back safely
- define the local and external truth conditions for calling the phase a success

## Deliverables

- `23-AUTH-SURFACE-CONTRACT.md`
- `23-SUPPORTED-FAMILY-MANIFEST.json`
- `23-GATE-RULES.md`

## Verification

- the contract names supported and unsupported families explicitly
- the contract is downstream-only and does not smuggle in new retrieval work
- the external gate condition is explicit and honest
- fallback safety remains defined outside the supported family boundary
