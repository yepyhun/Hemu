# Plan 02-03: Implement State Transitions And Maintenance Loops

## Objective

Operationalize the Phase 2 model with explicit admission, promotion, rejection, demotion, supersession, forgetting-by-demotion, and maintenance-loop behavior.

- maintenance loops exist for dedupe, conflict, supersession, stale provisional handling, and derived rebuild preparation
