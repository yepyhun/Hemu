# 08.1-01 Plan: Invariant Mapping And Boundary

## Goal

Define the exact hard rules to import from the legacy agent and map them onto existing Core2 boundaries before any implementation begins.

## Tasks

1. Review the legacy invariant set and select only the rules that are genuinely Core2-relevant.
2. Translate the selected rules into Core2 language and boundaries.
3. Write a small invariant catalog that states:
   - the invariant id
   - the Core2 rule
   - what kind of scenario proves or violates it
4. Explicitly list what is out of scope for this phase.

## Deliverables

- `08.1-INVARIANTS.md`
- `08.1-BOUNDARY.md`

## Verification

- The invariant list is short and concrete.
- Each invariant maps to an existing Core2 seam rather than inventing a second memory model.
- Boundary text excludes retrieval-ranking, activation maintenance, and broad legacy migration.
